# DK AgileBI 服务端设计说明

> 当前仓库为开源公开版，已移除 AI 相关功能与初始化脚本。

---

## 服务定位

当前版本聚焦于 BI 基础业务能力：

- 账号体系与权限边界
- 项目、数据集、组件、模板管理
- 文件上传与模板封面能力
- 企业字典、菜单与邮箱验证码

---

## 分层结构

- `routes/`：接口注册、参数校验、鉴权入口
- `services/`：业务实现、数据编排、响应结构处理
- `sql/`：数据库初始化脚本与增量脚本
- `config/`：运行配置、白名单、邮件与节点参数
- `utils/`：JWT、状态码、数据库封装、文件工具

---

## 设计原则

- 业务模块解耦，避免单点改动影响整条链路。
- 权限前置，先鉴权再查数，减少越权风险。
- 配置外置，数据库、邮件、OSS 等敏感内容不直接写入公开仓库。
- 文档与脚本保持最小公开集，只保留 BI 核心能力。

---

## 当前接口范围

- 用户与认证：登录、Token 校验、管理员用户管理、头像上传
- 项目管理：创建、详情、编辑、复制、发布、删除
- 数据集管理：新增、编辑、分页、权限控制
- 组件库：我的组件、公共组件、预览图上传、公开状态管理
- 模板中心：模板保存、详情、封面上传、复制转换
- 公共能力：OSS 访问信息、文件上传、企业字典、菜单、邮箱验证码

---

## 运行说明

启动前请自行提供以下配置：

- 数据库连接：`DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`
- 本地开发也可新建 `db/dbConfigDev.local.js` 保存本地数据库配置，该文件不会被 Git 提交
- 邮件服务：`SMTP_HOST`、`SMTP_PORT`、`SMTP_USER`、`SMTP_PASS`、`SMTP_FROM`
- 文件访问：`OSS_BUCKET_NAME`、`OSS_BUCKET_URL`

默认启动命令：

```bash
npm run nod:dev
```
